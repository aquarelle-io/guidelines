# Конфигурация 

Все настраиваемые данные помещаются в `config.inc.php`каталог верхнего уровня phpMyAdmin. Если этого файла не существует, обратитесь к разделу « [Установка](http://localhost/phpmyadmin/doc/html/setup.html#setup) », чтобы создать его. Этот файл должен содержать только параметры, которые вы хотите изменить, от их соответствующего значения по умолчанию `libraries/config.default.php`(этот файл не предназначен для изменений).

Смотрите также

 

[Примеры](http://localhost/phpmyadmin/doc/html/config.html#config-examples) примеров конфигураций

Если в вашем файле отсутствует директива, вы можете просто добавить еще одну строку с файлом. Этот файл предназначен для перезаписи значений по умолчанию; Если вы хотите использовать значение по умолчанию, здесь нет необходимости добавлять строку.

Вставляются параметры, относящиеся к дизайну (например, цвета) `themes/themename/layout.inc.php`. Вы также можете создать `config.footer.inc.php`и `config.header.inc.php`файлы, чтобы добавить код вашего сайта, который будет включен в начало и конец каждой страницы.

Заметка

 

Некоторые дистрибутивы (например, Debian или Ubuntu) хранить `config.inc.php`в `/etc/phpmyadmin`вместо пределах источников PhpMyAdmin.

Предупреждение

 

Пользователям[ Mac](http://localhost/phpmyadmin/doc/html/glossary.html#term-mac) следует отметить, что если вы используете версию до [Mac OS X](http://localhost/phpmyadmin/doc/html/glossary.html#term-mac-os-x) , PHP, похоже, не похожна символ конца строки линии[ Mac](http://localhost/phpmyadmin/doc/html/glossary.html#term-mac) (`\r`). Поэтому убедитесь, что вы выбрали опцию, которая позволяет использовать символ конца строки (`\n`) в текстовом редакторе перед сохранением скрипта, который вы изменили.

  Тип:целое числоЗначение по умолчанию:80Показывать предупреждение о неполных переводах с определенным порогом.


- `$cfg['SendErrorReports']`

  Тип:строкаЗначение по умолчанию:`'ask'`Устанавливает поведение по умолчанию для отчетов об ошибках JavaScript.Всякий раз, когда ошибка обнаруживается в выполнении JavaScript, отчет об ошибке может быть отправлен команде phpMyAdmin, если пользователь соглашается.Значение по умолчанию `'ask'`запрашивает пользователя каждый раз, когда появляется новый отчет об ошибке. Однако вы можете установить этот параметр для `'always'`отправки отчетов об ошибках без запроса подтверждения или вы можете установить его, `'never'`чтобы никогда не отправлять отчеты об ошибках.Эта директива доступна как в файле конфигурации, так и в настройках пользователей. Если лицо, ответственное за многопользовательскую установку, предпочитает отключить эту функцию для всех пользователей, значение `'never'`должно быть установлено, а[`$cfg['UserprefsDisallow'\]`](http://localhost/phpmyadmin/doc/html/config.html#cfg_UserprefsDisallow)Директива должна содержать `'SendErrorReports'`в одном из своих значений массива.


- `$cfg['ConsoleEnterExecutes']`

  Тип:логическийЗначение по умолчанию:ложныйУстановка этого параметра `true`позволяет пользователю выполнять запросы, нажимая Enter, вместо Ctrl + Enter. Новая строка может быть вставлена нажатием Shift + Enter.Поведение консоли можно временно изменить с помощью интерфейса настроек консоли.


- `$cfg['AllowThirdPartyFraming']`

  Тип:логическийЗначение по умолчанию:ложныйУстановка этого параметра `true`позволяет включить phpMyAdmin внутри фрейма и является потенциальным ядром безопасности, позволяющим атаковать межкадровыми сценариями или щелчками.

Д. В `libraries/config.default.php`нем есть только один раздел для определения сервера, однако вы можете поместить столько, сколько вам нужно `config.inc.php`, скопируйте этот блок или необходимые части (вам не нужно определять все настройки, только те Вам нужно изменить).Заметка [`$cfg['Servers'\]`](http://localhost/phpmyadmin/doc/html/config.html#cfg_Servers)Массив начинается с $ cfg ['Servers'] [1]. ## Каталоги загрузки / сохранения / импорта веб-сервера 

Если PHP работает в безопасном режиме, все каталоги должны принадлежать тому же пользователю, что и владелец скриптов phpMyAdmin.

Если каталог, в котором установлен phpMyAdmin, подлежит `open_basedir`ограничению, вам необходимо создать временный каталог в какой-либо директории, доступной интерпретатором PHP.

По соображениям безопасности все каталоги должны быть за пределами дерева, опубликованного веб-сервером. Если вы не можете избежать публикации этого каталога веб-сервером, ограничьте его доступом либо с помощью конфигурации веб-сервера (например, используя файлы .htaccess или web.config), либо поместите хотя бы пустой `index.html` файл, чтобы список каталогов не был возможен. Однако до тех пор, пока каталог доступен веб-серверу, злоумышленник может угадать имена файлов для загрузки файлов.

- `$cfg['UploadDir']`

  Тип:строкаЗначение по умолчанию:`''`Имя каталога, в котором файлы [SQL](http://localhost/phpmyadmin/doc/html/glossary.html#term-sql) были загружены другими способами, кроме phpMyAdmin (например, ftp). Эти файлы доступны в раскрывающемся списке, когда вы нажимаете имя базы данных или таблицы, а затем вкладку «Импорт».Если вам нужен другой каталог для каждого пользователя,% u будет заменен на имя пользователя.Обратите внимание, что имена файлов должны иметь суффикс «.sql» (или «.sql.bz2» или «.sql.gz», если включена поддержка сжатых форматов).Эта функция полезна, когда ваш файл слишком велик для загрузки через [HTTP](http://localhost/phpmyadmin/doc/html/glossary.html#term-http) или когда загрузка файлов отключена в PHP.Предупреждение Пожалуйста, ознакомьтесь с главой этой главы ( [каталоги загрузки / сохранения / импорта веб-сервера](http://localhost/phpmyadmin/doc/html/config.html#web-dirs) ) для получения инструкций по настройке этого каталога и способам его безопасного использования.Смотрите также См. Раздел [1.16. Я не могу загружать большие файлы дампа (проблемы с памятью, HTTP или таймаутом). ](http://localhost/phpmyadmin/doc/html/faq.html#faq1-16)Для альтернатив.


- `$cfg['SaveDir']`

  Тип:строкаЗначение по умолчанию:`''`Имя каталога, в котором можно сохранить дампы.Если вам нужен другой каталог для каждого пользователя,% u будет заменен на имя пользователя.Обратите внимание, что каталог должен существовать и должен быть доступен для записи для пользователя, на котором запущен веб-сервер.Предупреждение Пожалуйста, ознакомьтесь с главой этой главы ( [каталоги загрузки / сохранения / импорта веб-сервера](http://localhost/phpmyadmin/doc/html/config.html#web-dirs) ) для получения инструкций по настройке этого каталога и способам его безопасного использования.


- `$cfg['TempDir']`

  Тип:строкаЗначение по умолчанию:`''`Имя каталога, в котором могут храниться временные файлы.Это необходимо для импорта ESRI Shapefiles, см. [6.30 Импорт. ### Пример для ограниченного IP-адреса autologin 

Если вы хотите автоматически войти в систему при обращении к phpMyAdmin локально при запросе пароля при удаленном доступе, вы можете достичь этого, используя следующий фрагмент:

```
If  ( $ _SERVER [ "REMOTE_ADDR" ]  ==  "127.0.0.1" )  { 
    $ cfg [ 'Servers' ] [ $ i ] [ 'auth_type' ]  =  'config' ; 
    $ Cfg [ 'Servers' ] [ $ i ] [ 'user' ]  =  'root' ; 
    $ Cfg [ 'Servers' ] [ $ i ] [ 'password' ]  =  'yourpassword' ; 
}  Else  { 
    $ cfg [ 'Серверы' ] [ $ I ] [ 'auth_type' ]  =  'cookie' ; 
}

```

Заметка

 

Фильтрация на основе IP-адресов не является надежной через Интернет, используйте ее только для локального адреса.

### Пример использования нескольких серверов MySQL 

Вы можете настроить любое количество серверов, используя [`$cfg['Servers'\]`](http://localhost/phpmyadmin/doc/html/config.html#cfg_Servers), Следующий пример показывает два из них:

```
<? Php 
$ cfg [ 'blowfish_secret' ] = 'multiServerExample70518' ; 
// любая строка вашего выбора 
$ i  =  0 ;

$ I ++ ;  // сервер 1: 
$ cfg [ 'Серверы ] [ $ i ] [ ' auth_type ' ]  =  ' cookie ' ; 
$ Cfg [ 'Servers' ] [ $ i ] [ 'verbose' ]    =  'no1' ; 
$ Cfg [ 'Servers' ] [ $ i ] [ 'host' ]       =  'localhost' ; 
$ Cfg [ 'Servers' ] [ $ i ] [ 'extension' ]  =  'mysqli' ;


$ I ++ ;  // server 2: 
$ cfg [ 'Servers' ] [ $ i ] [ 'auth_type' ]  =  'cookie' ; 
$ Cfg [ 'Servers' ] [ $ i ] [ 'verbose' ]    =  'no2' ; 
$ Cfg [ 'Servers' ] [ $ i ] [ 'host' ]       =  'remote.host.addr' ; // или ip: '10 .9.8.1 ' 
// этот сервер должен разрешать удаленные клиенты, например, host 10.9.8.% 
// не только в mysql. 
Хост, но также и в конфигурации запуска $ cfg [ 'Servers' ] [ $ i ] [ 'extension' ]  =  'mysqli' ; 
// больше параметров для # 2 ...

// конец разделов сервера 
$ cfg [ 'ServerDefault' ]  =  0 ;  // выбор сервера при запуске

// дополнительные общие параметры ... 
?>

```

### Google Cloud SQL с SSL 

Чтобы подключиться к Google Could SQL, вам в настоящее время необходимо отключить проверку сертификата. Это вызвано выдачей certficate для CN, соответствующей вашему имени экземпляра, но вы подключаетесь к IP-адресу, и PHP пытается сопоставить эти два. С помощью verfication вы получите сообщение об ошибке:

```
Сертификат peer CN = `api-project-851612429544: pmatest 'не соответствует ожидаемому CN =` 8.8.8.8'

```

Предупреждение

 

С отключенной проверкой ваш трафик зашифрован, но вы открыты для людей в средних атаках.

Чтобы подключить phpMyAdmin к Google Cloud SQL, используя SSL, загрузите сертификаты клиента и сервера и скажите phpMyAdmin, чтобы использовать их:

```
// IP-адрес вашего экземпляра 
$ cfg [ 'Servers' ] [ $ i ] [ 'host' ]  =  '8.8.8.8' ; 
// Использовать SSL для соединения 
$ cfg [ 'Servers' ] [ $ i ] [ 'ssl' ]  =  true ; 
// Клиентский секретный ключ 
$ cfg [ 'Servers' ] [ $ i ] [ 'ssl_key' ]  =  '../client-key.pem' ; 
// Клиентский сертификат 
$ cfg [ 'Servers' ] [ $ i ] [ 'ssl_cert' ]  =  '../client-cert.pem' ; 
// Сертифицирующий сервер 
$ cfg [ 'Servers' ] [ $ i ] [ 'ssl_ca' ]  =  '../server-ca.pem' ; 
// Отключить проверку SSL (см. Выше примечание) 
$ cfg [ 'Servers' ] [ $ i ] [ 'ssl_verify' ]  =  false ; /server-ca.pem ' ; // Отключить проверку SSL (см. Выше примечание) $ cfg [ 'Servers' ] [ $ i ] [ 'ssl_verify' ] = false ; /server-ca.pem ' ; // Отключить проверку SSL (см. Выше примечание) $ cfg [ 'Servers' ] [ $ i ] [ 'ssl_verify' ] = false ;

```
